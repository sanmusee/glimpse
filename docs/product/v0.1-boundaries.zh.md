# Glimpse V0.1 产品边界简要记录

## 产品定位

Glimpse 是一个面向技术人员的 macOS AI SQL 客户端。它不是 DataGrip 替代品，也不是 ChatBI 或 BI 平台，而是一个 SQL editor-first 的本地开发者工具：AI 负责降低查表、写 SQL、改 SQL、修 SQL 的路径成本，开发者保留关键控制权。

V0.1 的核心气质是：在“自动”和“可控”之间取得平衡。

## V0.1 核心闭环

- 支持 MySQL/TiDB 直连。
- 支持多个数据库连接配置。
- 每个 Query Session 只绑定一个连接和一个默认 database/schema。
- 打开或连接时读取一次 catalog，后续由用户手动刷新。
- 用户输入自然语言需求后，Glimpse 可基于 catalog 自动发现候选表。
- 候选表必须可见、可调整，但不要求用户阻塞式确认。
- 支持 AI 生成 SQL。
- 支持基于当前 SQL 的自然语言修改。
- 支持用户手动执行只读 SQL。
- 支持展示查询结果。
- 支持 SQL 报错修复。
- 支持复制 SQL 和可见结果。
- 保存 Query Session、SQL 历史、SQL 草稿和 AI 对话历史。

## 执行与安全模式

- V0.1 只实现 Read-only Mode。
- 允许执行 `SELECT`、`WITH`、`EXPLAIN` 等只读语句。
- 阻止 `INSERT`、`UPDATE`、`DELETE`、`DROP`、`ALTER`、`TRUNCATE`、`CREATE` 等写操作或结构变更语句。
- AI 生成 SQL 后绝不自动执行，必须由用户手动点击执行。
- 对没有 `LIMIT`、可能长时间运行、结果集可能较大的查询，只警告，不自动改写 SQL，不强制追加 `LIMIT`。
- 未来计划支持 Data Modification Mode 和 Full Access Mode，但 V0.1 只做只读模式，并在设计上保留扩展性。

## 模型上下文边界

- V0.1 可以把默认 database/schema 的完整 catalog 元数据发送给用户配置的 OpenAI-compatible 模型，用于自动选表和 SQL 生成。
- 可发送 SQL 文本、数据库错误信息、数据库方言和相关 schema 给模型，用于 SQL 修复。
- 不发送数据库连接配置、密码、AI API key、SSH key 等敏感信息。
- 不发送样例数据。
- 不发送查询结果行。
- 全局只有一份 AI 配置，所有数据库连接共用。
- AI 生成、修改、修复采用流式输出。
- AI 调用失败时展示错误，由用户手动重试；V0.1 不做自动重试或模型 fallback。

## 本地数据与隐私

- V0.1 是纯本地 macOS App。
- 不做 Glimpse 账号。
- 不做云同步。
- 不做远端历史服务。
- SQLite 保存非秘密配置和工作历史，包括连接名称、host、port、默认 database/schema、Query Session、SQL 草稿、AI 对话历史、执行元信息和用户偏好。
- macOS Keychain 保存数据库密码、AI API key，以及未来可能支持的 SSH key。
- 查询结果集只在 UI 展示，不持久化保存。
- Query Session 只保存执行 SQL、时间戳、耗时、行数、成功/失败状态、错误信息等执行元信息。

## UI 与编辑体验

- V0.1 使用 SQL editor-first 布局。
- SQL 编辑器和查询结果是主工作区。
- AI 生成、自动选表、SQL 修改、报错修复和对话放在辅助面板。
- 不采用 chat-first 布局。
- 支持 Light、Dark、System 三种主题偏好。
- 直接进入主界面，通过空状态引导配置 AI 和第一个数据库连接；不做首次设置向导。
- SQL 编辑器基于 CodeMirror 6，支持 SQL 高亮和基础编辑体验。
- 支持手动格式化 SQL，但不自动格式化或静默改写。
- V0.1 不做表名/字段名自动补全。
- 结果表格为基础表格：列头、行号、横向滚动、复制单元格/行/可见结果。

## V0.1 明确不做

- 不做 DataGrip 替代品。
- 不做 ChatBI。
- 不做 BI 报表系统。
- 不做业务语义层或业务术语库。
- 不做 SQL 解释。
- 不做 SQL 自动补全。
- 不做 CSV 导出。
- 不做 Prompt 模板管理。
- 不做团队共享。
- 不做 SSH Tunnel、SSL 高级配置或堡垒机连接。
- 不做跨连接查询。
- 不做跨 database/schema 的 catalog 范围。
- 不持久化查询结果集。
